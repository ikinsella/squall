An Intuitive Tool for More Efficient Research

Why Squall?

Intuitive User Interface

Built with Flask and tested with users having a wide range of high throughput computing knowledge, Squall delivers a powerful tool for everyone to easily, quickly, and accurately perform large scale computations with the University of Wisconsin - Madison's HTCondor high throughput computing software.

Navigate the tabs at the top of the page to create the necessary components for jobs, the fundamental unit of execution in high throughput computing.

Robust Queriable Databases

The use of multiple types of databases allows Squall to more efficiently handle user information. The SQLite database stores user input to the web application and is known for it's simplicity and ease of use. Complementing SQLite, the MongoDB handling user submission and results files for HTCondor, as a more flexible database is needed to properly store unpredictable and variable data.

Distributed Computing Functionality

Squall provides high throughput functionality through UW - Madison's HTCondor. As a specialized workload management system for compute-intensive jobs, HTCondor harnesses the power of both dedicated servers and otherwise wasted CPU power from idle desktop workstations throughout the univeristy.

HTCondor can also extend it's computing 'grid' to encompass more computing resources from other universities and research institutions across the nation and across the globe.
